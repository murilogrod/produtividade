Sustentação Digital ao Negócio – Serviços Básicos

Nesta sessão serão descritos os serviços básicos necessários a comunicação primaria com o SIMTR para consulta de dados de clientes e criação e consulta das operações (dossiê de produto) bem como envio de atualizações e documentos.

Documentos de Clientes

GET /cadastro/v1/tipo-documento

  • Finalidade : Obter a lista de tipologia documental utilizada nas operações de sustentação digital ao negócio. Apesar de tender a uma estabilização, o retorno deste serviço pode sofrer alterações para adequar as necessidades de tipologia que são definidas pela área de gestão arquivistivca da CAIXA ou pela necessidade das operações de conformidade.

  • Corpo da Mensagem : Não se aplica.

  • Retorno :

  • identificador_tipo_documento: Código a ser utilizado no envio de novos documentos para correta identificação da tipologia.

  • nome_tipo_documento: Nome de identificação negocial do tipo de documento.

  • atributos_extracao: Caso existam definidos dados esperados para o documento, esta lista apresenta quais informações são esperadas, quais são obrigatórias e quais fazem parte de um mesmo grupo de dados, ou seja, caso seja enviado um elemento do grupo, todos os demais elementos deverão ser enviandos também, mesmo que não tenham cunho obrigatório. Para sistemas internos da CAIXA, no caso de envio de atributos do documento, o valor a ser utilizado como “chave” deverá ser o indicado no campo nome_documento.

POST /negocio/v1/dossie-cliente/{id}/documento

  • Finalidade : Inclui um documento encaminhados para armazenamento em um Dossiê de Cliente identificado pelo identificador único do mesmo junto ao SIMTR. Importante observar que se já existir um documento valido para novos negócios.

  • Corpo da Mensagem :

  • codigo_integracao: Informação fornecida pela GEBAN utilizada para identificar o canal de comunicação (informação depreciada pois o canal será identificado pelo seu clienteID do SSO;

  • tipo_documento: Identificador do tipo de documento, podendo ser obtido através do consumo do serviço: GET /cadastro/v1/tipo-documento

  • origem_documento: De forma a atender necessidades definidas no AD238 é necessario identificar a origem da midia em que o documento foi obtido podendo assumir:

  • A = Cópia Autenticada Adminstrativamente

  • C = Cópia Autenticada em Cartório

  • O = Original (Documento NatuDigital)

  • S = Cópia Simples

  • mimetype: Indicação do tipo de binario encaminhado utilizado para realizer o armazenamento de forma mais otimizada e viabilizar a manipulação do mesmo de forma adequada. Pode assumir um dos valores a seguir:
  • image/jpg, image/jpeg, application/pdf, image/png
  • binario: Conteudo do documento propriamente dito Deve ser enviada uma String no formato Base64.
  • Retorno :
  • Caso a inclusão seja bem sucedida, será retornado um código HTTP 204 (No Content) indicando que o documento foi armazenado e vinculado ao dossiê do cliente com sucesso.
