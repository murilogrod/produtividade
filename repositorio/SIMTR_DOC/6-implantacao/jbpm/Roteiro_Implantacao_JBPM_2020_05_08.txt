ROTEIRO DE IMPLANTAÇÃO (feito devido a incidente):

** HMP & PRD **
O módulo SIMTR JBPM possui uma comunicação interna entre os seus módulos que é feita via chamada HTTPS.
Desta forma, a aplicação precisa ter o protocolo HTTPS configurado e o KeyStore com as chaves pública e privada.

#Configuração do JBoss
    O KeyStore para o HTTPS deve ser configurado no standalone.xml dos servidores da seguinte forma:

	<security-realm name="JbpmSecurityRealm">
		<server-identities>
			<ssl>
				<keystore path="/opt/jbpm-server/standalone/configuration/keystore.jks" keystore-password="<senha_keystore>" alias="servercert"/>
			</ssl>
		</server-identities>
		<authentication>
			<truststore path="/opt/jbpm-server/standalone/configuration/keystore.jks" keystore-password="<senha_keystore>"/>
		</authentication>
	</security-realm>

    O protocolo HTTPS deve fazer referência ao security-realm configurado acima:

	<https-listener name="https" socket-binding="https" security-realm="JbpmSecurityRealm" enable-http2="true"/>

	Obs.: O alias da chave privada dentro do KeyStore deve ser a mesma do alias configurado no security-realm.

#Configuração do Servidor
	O servidor que atende ao SIMTR JBPM precisa ter uma configuração de loopback para própria URL no /etc/hosts:
	127.0.0.1       simtr-jbpm.caixa

	É necessário fazer o redirecionamento da porta 443 para a 8443, para que requisições que cheguem no servidor em HTTPS sejam recebidas pelo JBoss, que está respondendo em 8443.

	Esse redirecionamento é feito no firewall-cmd, utilizando os comandos abaixo:

#Habilitando o firewalld
systemctl start firewalld
systemctl enable firewalld

#Abrindo as portas padrões
firewall-cmd --add-port=10050/tcp # Zabbix Agent
firewall-cmd --add-port=10050/tcp # Zabbix Agent
firewall-cmd --add-port=8080/tcp  # JBoss

#Redirect da porta 443 para 8443
firewall-cmd --permanent --direct --add-rule ipv4 nat PREROUTING 0 -p tcp --dport 443 -j REDIRECT --to-ports 8443
firewall-cmd --permanent --direct --add-rule ipv4 nat OUTPUT 0 -p tcp --dport 443 -j REDIRECT --to-ports 8443 -o lo
firewall-cmd --reload
